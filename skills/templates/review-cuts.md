# 四刀审稿完整说明

> 当 aiwriter.md 主流程走到 Step 6 时 Read 此文件。
>
> 四刀流：**角度刀 → AI 味刀 → 废话刀 → 读者刀**。每刀必过，不打分，直接改。
> 前三刀是「作者刀」（自我审视），第四刀是「读者刀」（外部审视）——只有作者刀的稿子永远改不出读者读着是不是累。

## 性能约束：分节读 + 子代理化

**字数 ≥ 2500 字的文章必须遵守**：
- 前三刀（角度/AI 味/废话）：**用 Read 的 offset/limit 分节读**，每节单独过；禁止整篇 Read 后再四刀
- 第四刀（读者刀）：**用 Agent 工具调子代理**，子代理读全文 + 扮 3 类读者，主线只接收反应清单回报；详见末尾"读者刀子代理调用模板"

精炼版（< 2000 字）可以整篇 Read 一次过完前三刀，但读者刀建议仍走子代理。

---

## 第一刀：角度刀

**目的**：文章写完回头看，核心判断是否依然锋利？

1. 摘出**核心判断**（应来自 Step 2.5 门槛 C）
2. 重测「一刀测试」三条：
   - 可证伪：写下 X 发生则判断错误的条件
   - 反常识：60% 的读者第一反应是反向的？
   - 行动含义：读者能做一件具体不同的事？
3. **挡一次专家三刀**：随机找 Step 2.5 门槛 B 中的 1 位专家，他现在读完整篇是否会改口说"你其实没挡住我的刀"？

**修复动作**：
- 三条任一不过 → **开篇和结尾直接重写**
- 有节不支撑核心判断 → **砍掉这一节**
- 专家刀依然挡不住 → 在文章内补一段论据 + 收窄断言范围

---

## 第二刀：AI 味刀

**目的**：让稿子像一个具体的人写的，不像一个能写所有题材的 AI 写的。

1. **配额审计**：详见 `templates/quotas.md`
2. **风格抽测**（5 段样本）：开篇第 1 段 + 结尾最后 1 段 + 随机抽 3 段，对照「风格备忘卡」判断是不是该作家/记者腔的节奏。不像 → **整段重写**（不是换词，是换腔）
3. **AI 味 2.0 特征扫描**：详见 `templates/quotas.md`

---

## 第三刀：废话刀

**目的**：宁可短，不要凑。

1. **逐节 20% 削减**：标出每节最弱的 20%（注水句 / 重复前文 / 无新信息的过渡 / 空洞形容词段），直接删
2. **新信息密度检查**：每 150 字至少 1 个新信息（数据 / 具体案例 / 锋利判断）
3. **结尾审查**：总结式结尾 → 删除，改为金句 / 开放追问 / 直接结束在最锋利判断句上

---

## 第四刀：读者刀

**目的**：前三刀是作者自检，读者刀强制走出作者视角。

### 1. 挑 3 类目标读者

基于 Step 0 第 4 题"目标读者"画像，AI 各扮 1 个**具体的人**（不是"中年男性"——是"38 岁互联网中层，每天通勤 1 小时看公众号，刷到 3 段没钩子就关"）：

- **读者 A — 核心共鸣型**：完全契合目标的核心读者（最容易转发的那一类）
- **读者 B — 怀疑反驳型**：擦边目标但有反对意见的读者（最有可能在评论区开炮的那一类）
- **读者 C — 路人耐心薄**：被标题吸引点进来但不属于核心目标的读者（决定打开率→读完率转化的那一类）

### 2. 每位读者读全文，标 5 类反应点

| 反应 | 含义 | 修复动作 |
|------|------|---------|
| 🚪 想关掉 | 这段开始想滑走 | **重写该段**（往前提锋利点 / 删过渡 / 换具体场景） |
| 🪨 想反驳 | 这句感觉作者偷换概念或越线 | **补一句论据**回应 / 收窄断言 / 注明对立成立条件 |
| 📤 想转发 | 想截图给朋友 | 保留并 check 是否可升级为加粗金句（按 quotas） |
| ✏️ 想划线 | 想划下来留着 | 同上 |
| ⏭️ 想跳过 | 觉得是凑字数可略过 | **直接砍** |

### 3. 优先级

想关掉 > 想反驳 > 想跳过 > 想转发/想划线。前三类必修，后两类是收益项。

### 4. 粘连性 check（修完后跑一遍）

- 砍段/重写段后，前段末尾和后段开头是否还接得上？不接 → **补 1 句桥**
- 开篇钩子承诺的"核心判断的一半"或"反常识点"，在结尾是否被回收？没回收 → 结尾补 1-2 句明示
- 加粗金句配额是否仍在硬配额内？超 → 砍最弱的

### 5. 简报格式

```
✂️ 读者刀（共改 X 处）：
  - 读者 A 反应：想转发 N 处、想划线 M 处（皆保留）
  - 读者 B 反应：想反驳 K 处 → 补论据 K 处
  - 读者 C 反应：想关掉 J 处（前 3 段集中）→ 重写开篇 + J-1 段
  - 想跳过 L 处 → 砍 L 段
  - 桥句补 P 处，结尾回收钩子 1 处
```

### 读者刀子代理调用模板

```
Agent(
  description="读者刀审稿 - 3类读者反应分析",
  subagent_type="general-purpose",
  prompt="""
你是文章审稿员。Read 文件 {article.md 完整路径}（整篇读完）。

文章背景：
- 标题：{标题}
- 平台：{微信公众号/小红书}
- 目标读者画像：{从 Step 0 第 4 题来}
- 核心判断：{从 Step 2.5 门槛 C 来}

任务：扮演 3 类具体读者各读一遍，给每段标反应点。

读者画像（必须用具体的人，不是泛泛描述）：
- 读者 A 核心共鸣型：[在此具体描述]
- 读者 B 怀疑反驳型：[在此具体描述]  
- 读者 C 路人耐心薄：[在此具体描述]

每位读者读完全文后，在反馈中标出 5 类反应：
🚪 想关掉、🪨 想反驳、📤 想转发、✏️ 想划线、⏭️ 想跳过

输出格式（仅返回此清单，不要重复全文）：

【读者 A 反应】
- 段落 N（"开头一句..."）：📤 想转发 — 理由
- ...

【读者 B 反应】
- 段落 M（"开头一句..."）：🪨 想反驳 — 理由
- ...

【读者 C 反应】
- 段落 K（"开头一句..."）：🚪 想关掉 — 理由
- ...

【综合建议】（≤200 字，给主代理执行）
- 必修高优先级 N 处：分别是 [段落] - [建议改法]
"""
)
```

主代理只接收子代理回报的清单（约 500-800 字），不接收文章原文。然后执行修复动作。

## 执行逻辑（按 Step 0 第 9 题"审稿偏好"分支）

**分支 A：自动四刀审稿并修复（默认）**
1. 按顺序执行四刀，每刀内部逐项核对并直接改文
2. 改完后用一段话简要汇报（**不给分**）：
   ```
   📝 四刀审稿完成：
   ✂️ 角度刀：核心判断通过 / 砍掉第 3 节 / 补专家 A 刀回应段
   ✂️ AI 味刀：删比喻 2 处、删加粗 3 处、删段首路标词 5 处
   ✂️ 废话刀：每节削减 20%，全文从 2100 字精简到 1650 字
   ✂️ 读者刀：读者 C 想关掉 2 处 → 重写开篇；想反驳 1 处 → 补论据；砍 1 段；补桥句 2 处
   ```
3. 直接进入 Step 7

**分支 B：审稿后用户决定**
- 四刀每刀单独出一份诊断清单（诊断 + 建议改写）
- 用户选择"全部接受 / 按刀接受 / 按条接受 / 手动改"

**分支 C：跳过审稿**
- 直接进入 Step 7
